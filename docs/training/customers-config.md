# Customers + Config

## Objetivo

Validar alta y detalle de cliente, documentos y configuración operativa que alimenta el flujo real de crédito.

## Flujo de demo

1. Abrir listado de clientes.
2. Entrar al detalle.
3. Subir documento y verificar categoría/visibilidad.
4. Desactivar y reactivar cliente.
5. Abrir Configuración.
6. Crear o editar método de pago.
7. Revisar políticas de tasa y mora.

## Checklist QA

- Formularios con validaciones visibles.
- Documentos aceptan formatos permitidos.
- Métodos de pago activan/desactivan sin recargar roto.
- Roles y permisos cargan usuarios reales.

## Storyboard

1. Listado de clientes.
2. Documento en detalle de cliente.
3. Cambio de estado del cliente.
4. Alta o edición de método de pago.
5. Revisión de políticas vigentes.
